# Building a Legacy with AI: The Journey to Google SRE/SWE

In the world of technology, leaving a lasting legacy is more than just creating software; it's about building systems that stand the test of time and inspire others. This blog chronicles my journey to achieve this goal by leveraging AI-driven automation, while also preparing for a potential role at Google as an SRE or SWE engineer.

## Problem

As a legacy designer, my primary goal is to build robust systems that not only solve current problems but also serve as a foundation for future innovations. My roadmap is ambitious, aiming to create an integrated platform that automates knowledge management, financial tracking, and personal health monitoring. Additionally, I am preparing to apply for senior positions at Google by showcasing my technical prowess and reliability.

## Approach

To tackle this multifaceted project, I have structured it into several phases:

1. **Base Camp & Identity**: Establishing a strong online presence and foundational systems.
2. **Data Foundation & Automation**: Building an integrated backend and data storage system.
3. **Finance & Memory Modules**: Developing core functionalities for financial tracking and personal health monitoring.
4. **Branding & SRE Audit**: Ensuring system reliability and preparing comprehensive documentation.
5. **Global Target Attacking**: Preparing for a senior role at Google by demonstrating my work.

### Phase 1: Base Camp & Identity

The first phase focused on setting up a strong online presence and foundational systems. I developed an MVP of the official brand website using Next.js, which serves as a central hub for all my projects. I also set up GitHub Actions for CI/CD, ensuring that the deployment process is automated and reliable. Additionally, I published my legacy declaration on the website to establish a personal brand identity.

### Phase 1.5: Brand Site 'Live' Dynamic Integration and SRE Automation

In this phase, I am enhancing the brand website to dynamically integrate with backend systems and automate SRE processes. For instance, I have integrated my tech blog's omnichannel automation system, where Git commit messages are parsed and sent to both the Next.js website and Dev.to. This ensures that my latest content is always up-to-date across multiple platforms.

### Phase 2: Legacy_Core Engine Development (Java/Spring Boot)

The core of my system is the `Legacy_Core` engine, which serves as a central hub for all data management. I am developing this using Spring Boot with strict security protocols to ensure data integrity and reliability. The engine is designed to handle various domains such as assets, memories, and values, with a robust API interface for seamless integration.

### Phase 3: Client Edge Applications (Flutter App)

In this phase, I plan to develop client edge applications using Flutter. These apps will allow users to interact with the system on a daily basis, providing real-time data and insights. The first application in this series is the SFX Life-Log, which will integrate with Samsung Health Connect API to collect biometric data and provide personalized health insights.

## Implementation

### Tech Blog Omnichannel Automation

To automate the distribution of my tech blog posts, I implemented a system where Git commit messages trigger an AI agent to parse and format the content. This content is then automatically deployed to both my Next.js website and Dev.to, ensuring that my latest insights are always accessible.

### Spring Boot Backend Integration

For the `Legacy_Core` engine, I have implemented a secure authentication system using Spring Security and JWT tokens. This ensures that only authorized users can access sensitive data. Additionally, I have integrated health check endpoints to monitor the system's reliability in real-time.

### Flutter Application Development

For the SFX Life-Log app, I am leveraging Flutter's cross-platform capabilities to build a user-friendly interface. The app will integrate with the `Legacy_Core` engine to collect and display biometric data, providing users with personalized insights into their health.

## Lessons Learned

### Importance of Automation

One of the key lessons I have learned is the importance of automation in software development. By automating tasks such as deployment and content distribution, I have been able to focus on more critical aspects of the project. This has not only improved efficiency but also ensured consistency across different platforms.

### The Power of AI

AI-driven automation has been a game-changer in my project. By leveraging AI agents to parse and format content, I have been able to streamline the development process and ensure that my latest insights are always up-to-date. This has also allowed me to focus on more complex aspects of the project.

### Continuous Learning

Finally, I have learned that continuous learning is essential in this field. As technology evolves rapidly, it's crucial to stay up-to-date with the latest tools and techniques. This has been particularly important in my journey to become a legacy designer, as I have had to adapt to new tools and technologies to build robust systems.

## Conclusion

Building a legacy with AI is a challenging but rewarding journey. By leveraging automation, AI-driven tools, and continuous learning, I have been able to develop a robust system that serves as a foundation for future innovations. As I continue on this journey, I am excited to see where it will take me and what new challenges I will face.
