# Phase 1.5: 브랜드 웹사이트의 동적 연동 및 SRE 자동화 구현

# Phase 1.5: 브랜드 웹사이트의 동적 연동 및 SRE 자동화 구현

## 문제
현재 우리의 브랜드 웹사이트는 정적 콘텐츠로 구성되어 있습니다. 그러나 우리의 비전은 실시간 데이터 업데이트와 사용자 경험 향상에 중점을 둔 동적 웹사이트를 구축하는 것입니다. 이를 위해, Next.js 기반의 브랜드 웹사이트와 백엔드 시스템을 더 깊이 통합하여 실시간 데이터 업데이트와 자동화 프로세스를 구축해야 합니다. 또한, 모달 창과 실서비스 리소스의 동적 연동을 통해 사용자 경험을 향상시키는 작업도 진행해야 합니다.

## 접근
우리의 접근 방식은 두 가지 주요 부분으로 나뉩니다. 첫째, Tech Blog의 자동화 연계를 통해 Git Commit 파싱 스크립트가 단순히 외부 블로그에 송출하는 데 그치지 않고, 로컬 AI(LM Studio)의 4단계 추론(Phase Analysis)을 거쳐 완벽한 영문 서사로 Next.js 웹사이트 방어막(`drafts/`) 및 Dev.to(Canonical SEO)로 동시 배포되는 시스템을 구축합니다. 둘째, 모달 및 실서비스 리소스의 동적 연동을 통해 Google Play Store(Life-Log) 및 Medium(SRE Blog) 링크를 메인 페이지 카드에 직접 결속하여 즉시 접근 가능한 베이스캠프 기능을 강화합니다.

## 구현
### Tech Blog 자동화 연계
Tech Blog의 자동화 연계는 로컬 AI(LM Studio)와 Next.js 웹사이트를 연결하여 Git Commit 파싱 스크립트가 외부 블로그에 송출하는 것 이상의 기능을 제공합니다. 로컬 AI(LM Studio)는 4단계 추론(Phase Analysis)을 거쳐 완벽한 영문 서사로 Next.js 웹사이트 방어막(`drafts/`) 및 Dev.to(Canonical SEO)로 동시 배포되는 시스템을 구축합니다. 상세 아키텍처 및 코어 워크플로우는 `01-1-phase1.5-sre-automation-details.md` 하위 문서 참조.

### 모달 및 실서비스 리소스 연동
모달 및 실서비스 리소스의 동적 연동은 Google Play Store(Life-Log) 및 Medium(SRE Blog) 링크를 메인 페이지 카드에 직접 결속하여 즉시 접근 가능한 베이스캠프 기능을 강화합니다. 이를 위해, Next.js 웹사이트와 백엔드 시스템을 더 깊이 통합하여 실시간 데이터 업데이트와 자동화 프로세스를 구축해야 합니다.

## 배운 점
이 단계에서 가장 중요한 배운 점은, 동적 웹사이트 구축을 위해 Next.js와 백엔드 시스템의 깊은 통합이 필요하다는 것입니다. 또한, 로컬 AI(LM Studio)와의 연결은 Tech Blog의 자동화 연계를 통해 외부 블로그에 송출하는 것 이상의 기능을 제공할 수 있습니다. 이러한 접근 방식은 사용자 경험 향상과 실시간 데이터 업데이트를 가능하게 합니다.

## 결론
Phase 1.5에서는 브랜드 웹사이트의 동적 연동 및 SRE 자동화를 목표로 하여, Next.js와 백엔드 시스템의 깊은 통합을 통해 실시간 데이터 업데이트와 자동화 프로세스를 구축하고, 모달 및 실서비스 리소스의 동적 연동을 통해 사용자 경험을 향상시켰습니다. 이러한 작업은 우리의 비전인 실시간 데이터 업데이트와 사용자 경험 향상에 중점을 둔 동적 웹사이트 구축을 가능하게 합니다.
