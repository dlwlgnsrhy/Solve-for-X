# Sovereign Issue Git Workflow 스킬 사용자 가이드

본 폴더는 AI 코딩 에이전트에게 **로컬 이슈 트래킹 및 Conventional Commits 무결성 협업 프로토콜**을 주입하여, 안전하고 완벽한 형상 관리를 오토파일럿으로 수행하도록 지시하는 영구 스킬 패키지입니다.

---

## 🚀 1. 이 스킬은 무엇인가요?

우리가 함께 빌드한 고도화된 형상 관리 체계(마일스톤 수립 -> 개별 이슈 문서 쪼개기 -> 린트 훅 충족 Conventional Commits 분할 발행 -> 메인 병합 및 린업)를 **에이전트가 통째로 기억하여 수행하도록 행동 강령을 박제한 커스텀 스킬**입니다.

에이전트는 이 폴더 내의 `SKILL.md`를 섭취(Ingest)하는 순간, 개발 작업을 시작할 때 이 규칙을 최우선 탑재하여 움직입니다.

---

## 🛠️ 2. 사용 방법 (에이전트 호출 요령)

앞으로 새로운 프로젝트, 새로운 기능 개발 또는 앱 이식 작업을 에이전트와 페어 프로그래밍으로 착수하실 때, **대화의 시작 프롬프트**에 다음과 같이 적어주시면 됩니다:

> **"docs/guidelines/sovereign-issue-git-workflow/ SKILL.md 규칙에 따라서, 지금 요청하는 [새로운 기능 설명] 기능을 개발해줘."**

이 지시를 받으면 에이전트는:
1. 즉시 `docs/milestones/` 아래에 로컬 이슈 마크다운 문서들을 먼저 생성합니다.
2. 각 세부 구현 단계를 쪼갠 구현 계획서(implementation_plan.md)를 올려 사용자님의 결재를 구합니다.
3. 깃 커밋을 날릴 때 커밋 메시지 규칙(Type/Scope 필수 탑재)과 `(resolves #X)` 연결 고리를 매 작업 단위마다 자동으로 엮어서 촘촘하게 저장해 나갑니다.
4. 작업 완수 후 메인 브랜치 병합(`git merge`) 및 피처 브랜치 삭제 린업까지 깔끔하게 끝마치고 보고합니다.

---

## 📌 3. 보관 및 확장 요령

* 본 스킬은 `architecture/docs/guidelines/` 아래에 안전하게 보존되며 프로젝트 코어의 일부로 작동합니다.
* 형상 관리 린트 규칙(예: 새로운 Commit Type 추가 등)이 변경될 경우, 이 폴더의 `SKILL.md` 내용만 수정해주시면 에이전트는 수정된 규칙을 즉시 학습하여 새로운 기준대로 작동하게 됩니다.
