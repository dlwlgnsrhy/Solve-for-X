# [비즈니스 팩토리 편] 1장. 처음부터 코딩하지 마라: '설계도(Blueprint)'와 코어 템플릿의 비밀
**부제: 1인 개발 공장의 생산 라인 표준화: Flutter, Next.js, Supabase**

[기술 철학 편]에서 우리는 거대 클라우드 시대에 데이터 주권을 수호하고 인간 고유성을 영구히 보존하기 위한 장엄한 비전들을 나누었습니다. 이 심오한 철학을 마음에 담았을 때, 현실적인 벽이 우리 앞을 가로막습니다.

"이 많은 앱(Imjong Care, Legacy Vault, Origin 등)을 1인 개발자가 대체 언제 다 만들고 관리한단 말인가?"

철학이 아무리 아름다워도 현실에 프로덕트로 나타나지 못하면 그것은 공상에 불과합니다. 1인 개발자에게 시간과 에너지는 극도로 한정된 자원입니다. 

이를 해결하기 위해 제가 고안한 해결책이 바로 **'비즈니스 팩토리(Business Factory)'**—즉, 무한한 철학적 아이디어를 빠르고 일관되게 찍어내는 무인 앱 공장을 세우는 것이었습니다. 그리고 그 공장의 시작은 바로 **'생산 라인의 표준화'**에 있었습니다.

---

### 🕸️ 매번 새로 짜는 코딩은 죄악이다

많은 개발자들이 새로운 프로젝트를 시작할 때 엄청난 설렘을 느낍니다. 하지만 설렘도 잠시, 매번 반복되는 지루한 작업에 마주하게 됩니다.
*   디렉토리 구조 설계 (Clean Architecture 세팅)
*   사용자 로그인 및 회원가입(Auth) 흐름 구현
*   다크모드 지원 및 기본 스타일 가이드 적용
*   상태 관리 라이브러리 연동 및 라우터 설정

이런 보일러플레이트(Boilerplate) 코드를 매번 맨땅에서 새로 짜는 것은 1인 창업가에게 치명적인 시간 낭비입니다. 공장을 돌리려면 나사 하나까지 매번 깎아 쓰는 것이 아니라, 표준화된 규격의 기성 부품을 조립해야 합니다.

### ⚙️ 생산 라인의 표준 규격: Flutter, Next.js, Supabase

저는 Solve-for-X 생태계의 모든 서비스를 빠르게 벼려내기 위해 다음과 같이 기술 스택을 완벽하게 표준화했습니다.

1.  **모바일 앱 (Mobile): Flutter**
    *   크로스 플랫폼 앱 딜리버리의 최강자. 단 하나의 코드로 iOS와 Android를 동시에 정복합니다.
    *   Riverpod(상태 관리)과 Clean Architecture를 결합한 'Core Template'을 미리 구축해 두었습니다. 디렉토리 구조는 물론이고, 공통 UI 컴포넌트까지 완벽하게 규격화되어 있습니다.
2.  **웹 & 랜딩 페이지 (Web): Next.js / React**
    *   유저 관리 콘솔, 서비스 소개 랜딩 페이지, 어드민 패널은 최고의 웹 프레임워크인 Next.js로 통일했습니다.
3.  **백엔드 서비스 (BaaS): Supabase & Firebase**
    *   서버 인프라를 구축하고 DB API를 직접 설계하는 수고를 덜기 위해 백엔드를 모듈화했습니다. 특히 Supabase는 온디바이스 DB 설계와 강력하게 호환되어 로컬 지향적인 비전에 부합합니다.

### 📜 '설계도(Blueprint)'에 따라 조립되는 템플릿 방식

이제 새로운 앱 아이디어가 떠오르면 저는 키보드에 손을 얹지 않습니다. 대신 미리 정의해 둔 `design_tokens.json`과 `common_models.json` 설정 파일을 작성합니다.
*   `design_tokens.json`에 브랜드 고유 컬러, 폰트, 테마 정의.
*   `common_models.json`에 유저 정보, 스탬프, 로그 등 데이터 스키마 정의.

준비된 코어 템플릿에 이 설정 파일들을 주입하면, 기본적인 UI 뼈대와 인증 계층, 데이터 연동 코드가 자동으로 설정되고 컴파일됩니다. 프로젝트 시작 직후 몇 분 안에 바로 비즈니스 로직(X)만 집중하여 코딩할 수 있는 완벽한 상태가 갖춰지는 것이죠.

### 🤖 AI 에이전트에게 최고의 윤활유가 되는 '표준화'

중요한 점은, 이 생산 라인의 표준화가 인간 개발자뿐만 아니라 **Part 1에서 소개한 AI 에이전트(Hermes와 OpenAgent)에게 강력한 힘을 실어준다는 사실**입니다.

AI 에이전트가 코딩을 하다가 무한 루프에 빠지거나 엉뚱한 코드를 빌드하는 가장 큰 원인은 '예측 불가능한 코드베이스 환경' 때문입니다. 아키텍처가 제각각이면 AI는 컨텍스트를 파악하는 데 지능을 낭비하고 실수를 연발합니다.

하지만 모든 앱이 동일한 Core Template과 설계도(Blueprint) 하에 엄격하게 규격화되어 있다면 어떨까요?
Hermes와 OpenAgent는 파일 구조와 API 패턴을 100% 예측할 수 있습니다. AI가 코드를 생성하고 검증하는 성공률이 기하급수적으로 높아지는 것입니다. 인간이 마련해 둔 표준화된 가이드레일 위에서 AI 에이전트는 날개를 달고 안전하게 자동화 라인을 가동하게 됩니다.
