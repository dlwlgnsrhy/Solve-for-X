# TS-0009: 자동화 파이프라인 안정성 강화 및 필터링 고도화 (2026-04-19)

## 1. 개요
2026년 4월 18일~19일 사이 발생한 자동화 파이프라인의 3가지 주요 장애 상황을 분석하고 해결책을 적용함.

## 2. 발생 장애 및 원인 분석

### 가. 라이브 프리뷰 502 Bad Gateway
- **상황**: 텔레그램으로 전송된 실물 테스트 링크 클릭 시 502 에러 발생.
- **원인**: `visual_tester/main.py`가 웹 서버 기동 시 설치되지 않은 `playwright` 라이브러리를 임포트하다가 크래시 발생. 터널은 열려 있으나 백엔드 서버가 죽어 502 반환.
- **해결**: `playwright` 임포트를 스크린샷 캡처 함수 내부로 이동(Lazy Import)하여 서버 단독 기동 시 의존성 충돌 제거.

### 나. 저녁 회고 메시지 도배 (Spam Loop)
- **상황**: 자율 수정 에이전트가 API를 테스트할 때마다 텔레그램으로 "저녁 회고 제안"이 반복 전송됨.
- **원인**: API 엔드포인트가 `daily_planner`를 트리거할 때, 밤 시간대면 항상 회고 제안을 생성하고 전송함. 중복 체크 로직 부재.
- **해결**: `NotionClient` 및 `daily_planner`에 중복 블록 감지 로직 추가. 동일 날짜에 이미 회고 블록이 존재하면 생성 및 전송 생략.

### 다. 뉴스 큐레이터 0건 선별 (Zero-Hit)
- **상황**: 관심 기술 기사가 많음에도 불구하고 "선별된 기사 없음" 보고.
- **원인**: 
    1. 주말 기술 블로그 발행량 급감.
    2. 엄격한 24시간 수집 범위 (`COLLECT_HOURS = 24`).
    3. 로컬 LLM(Qwen 14B)의 모호한 Yes/No 판단 (페르소나 부재).
- **해결**:
    1. **Weekend Logic**: 주말(토/일)에는 자동으로 수집 범위를 **72시간**으로 확대.
    2. **Persona 부여**: 로컬 LLM에 시니어 엔지니어 큐레이터 페르소나 주입.
    3. **Observability**: 필터링 과정 로그([PASS]/[SKIP])를 상세하게 남기도록 개선.

## 3. 결과 및 교훈
- **결과**: 주말 뉴스 브리핑에서 5건의 양질 기사 선별 성공. 502 에러 및 메시지 도배 현상 제거 완료.
- **교훈**: 
    1. 자동화 루프 내에서의 API 테스트는 의도치 않은 트리거(Side Effect)를 일으킬 수 있으므로 멱등성(Idempotency) 확보가 필수적임.
    2. 주말과 평일의 데이터 특성이 다르므로 가변적인 수집 정책이 필요함.
