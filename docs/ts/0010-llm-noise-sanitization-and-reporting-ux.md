# TS-0010: LLM 추론 과정(CoT) 유출 및 보고 가독성 저하 해결

## 🚨 문제 상황 (Problem)
- **AI 노이즈 유출**: `daily_planner`, `weekly_planner` 등에서 LLM(Qwen 35B)이 프롬프트 지시를 무시하고 내부 추론 과정(Thinking process, Analyze User Input 등)을 노션 본문에 그대로 출력함.
- **보고 피로도**: 텔레그램으로 전송되는 블로그 초안 및 주간 계획 보고서가 지나치게 장황하고, AI의 자문자답이 섞여 있어 관리자(사용자)의 가독성을 저해함.
- **출력 끊김**: 추론 과정이 길어지면서 `max_tokens` 제한에 걸려 정작 중요한 템플릿 본문이 잘리거나 미출력되는 현상 발생.

## 🔍 원인은 무엇인가? (Root Cause)
1. **모델의 Verbosity**: 추론 능력이 강화된 모델(Qwen 35B 등)은 지시사항보다 자신의 추론 과정을 우선 출력하려는 경향이 강함.
2. **토큰 부족**: 1200~2000 토큰 제한 환경에서 추론 과정이 전체 토큰의 80% 이상을 점유하여 본문 출력이 잘림.
3. **Regex 고립성 부족**: 초기 필터링 로직이 문서 상단의 특정 키워드만 탐지하여, 중간에 섞여 들어오는 "1. Analyze..." 등의 리스트형 추론을 거르지 못함.

## 🛠️ 해결 방법 (Solution)

### 1. 전역 시스템 정제기 (Last-Match Anchor Sanitizer)
- `LLMClient` 코드 레벨에서 응답을 반환하기 전 **물리적 필터링**을 수행함.
- **앵커 탐지**: 사용자의 정규 템플릿 시작점(`###`, `- ` 등)을 하드코딩된 앵커로 지정.
- **후방 탐색(Last-match)**: 문서 내에서 필터링 앵커가 여러 번 나타날 경우(AI가 계획 단계에서 템플릿을 복제한 경우 등), **가장 마지막에 나타난 앵커**를 기준으로 그 이전의 모든 내용을 강제 삭제함.
- 이로써 AI가 앞에 무슨 말을 하든 무조건 깨끗한 템플릿부터 시작하는 결과물을 보장함.

### 2. 인프라 파라미터 최적화
- **Token 상향**: `max_tokens`를 3500-4000 수준으로 대폭 늘려 AI가 충분히 추론하고도 본문을 남길 수 있는 공간 확보.
- **Concise Prompting**: 시스템 프롬프트 상단에 "추론 과정은 최대한 짧게 핵심만 하라"는 메타 지침 삽입.

### 3. 보고서 UX 개편 (Executive Summary)
- 텔레그램 전송 로직을 수정하여 AI의 원문을 직접 덤프하지 않고, **[제목 / 요약 / 경로 / 다음 액션]**만 추출하여 전송하는 브리핑 체계 구축.

## ✅ 결과 및 교훈 (Outcome & Lessons)
- **가독성 100% 확보**: 노션 본문에서 모든 AI 노이즈가 제거되고 템플릿만 남는 것을 API 감사를 통해 확인.
- **관리 효율 향상**: 텔레그램 보고서가 300자 내외로 압축되어 모바일에서도 한눈에 파악 가능.
- **교훈**: AI에게 "출력하지 마라"고 지시하는 것보다, 시스템 레벨에서 **"출력된 것을 도려내는"** 하드 가드레일이 훨씬 강력하고 확실하다.
