# SFX Imjong Care MVP Blueprint

## 1. Core Philosophy & Target
- **Slogan:** "Living Well = Dying Well" (오늘 하루의 끝을 기획함으로써, 내일을 더 밀도 있게 산다)
- **Target Audience:** MZ 세대, 번아웃을 겪는 직장인, 자기 객관화와 '갓생'을 추구하는 유저.
- **Vibe & Tone:** 우울하고 무거운 '죽음'이 아닙니다. 트렌디하고, 힙하며(Neon/Dark Mode), '인생 네컷'처럼 내 인생의 한 챕터를 아름답게 갈무리하는 **'스타일리시한 인생 마감 앱'**.

## 2. Core Features (MVP) - 딱 3가지만 시장에 던집니다.

### Feature 1: "Today's Ending" (오늘의 마감)
- **기능:** 매일 밤 알림이 오면, 단 한 줄의 질문에 답합니다. (예: "오늘이 마지막 날이라면, 오늘 하루는 몇 점짜리였나요?", "오늘 가장 후회되는 15분은 언제인가요?")
- **가치:** 죽음을 가정함으로써 역설적으로 오늘 하루의 메타인지를 극대화합니다.

### Feature 2: "Digital Will Card" (내 가치관 명함)
- **기능:** 앱 가입 시, 내가 죽을 때 남기고 싶은 3가지 가치(예: 자유, 사랑, 호기심)와 한 줄 유언을 작성합니다. 이를 인스타그래머블한 '디지털 유서 명함(3D Card UI)'으로 만들어줍니다.
- **가치:** 공유 가능한 포맷으로 만들어 바이럴 루프(Viral Loop)를 생성합니다.

### Feature 3: "Time Capsule Vault" (레거시 볼트)
- **기능:** 특정인(가족, 연인, 친구)에게 남기는 영상이나 텍스트를 저장합니다. 지정한 기한(또는 로그인 비활성화 기간)이 지나면 발송되는 형태의 컨셉 UI만 MVP에 포함합니다.
- **가치:** Dying Well의 가장 실용적인 기능(유산/메시지 상속)을 제공합니다.

## 3. 개발 및 검증 전략 (App Factory 활용)
- **Tech Stack:** Flutter (Riverpod, Freezed), Firebase (간단한 Auth 및 DB).
- **개발 방식:** 이 문서의 기획안을 바탕으로 AI App Factory (6-Wave 파이프라인)에 던져서 UI 뼈대와 로직을 1주 내에 자동 생성합니다.
- **지훈님의 역할:** 코딩이 아닌, "디지털 유서 카드가 얼마나 예쁘게 렌더링되는가?", "질문이 얼마나 가슴을 찌르는가?"에 집중하여 프롬프트를 조율합니다.
