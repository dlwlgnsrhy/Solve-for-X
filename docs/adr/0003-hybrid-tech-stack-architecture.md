# 3. 하이브리드 멀티플랫폼 아키텍처 (Next.js + Flutter + Java/Spring) 채택

Date: 2026-03-29

## Status
Accepted

## Context
- Solve-for-X 프로젝트는 다수의 모듈(Brand Web, Life-Log, Finance App)이 하나의 가치관("The Tech of Human Dignity") 아래 연결되는 거대한 생태계입니다.
- 레거시 설계자(사용자)는 다수의 앱을 만들며 생산성이 검증된 **Flutter(Dart)** 경험을 보유하고 있습니다.
- "어떤 프레임워크가 살아남을지 모른다"는 불확실성에 대비하여, SRE/Architecture 관점에서 미래 생존율(Longevity)이 높고 의존성이 철저히 분리된 설계를 채택해야 할 필요성이 대두되었습니다.
- SEO(검색 최적화)가 중요하고 즉각적인 로딩이 필요한 퍼블릭 '웹(Brand Site)' 영역과, 장치 제어 및 네이티브 퍼포먼스가 중요한 '앱' 영역의 요구사항은 완전히 상충합니다.
- (추가 반영) 글로벌 엔터프라이즈 환경에서의 견고한 아키텍처 경험과 이력서 스펙 구축(SRE/SWE)을 목적으로, 백엔드 기술 스택으로 **Java/Spring Boot**를 채택하기로 협의했습니다. 

## Decision
- 단일 프레임워크 락인(Lock-in)을 피하고 목적에 맞는 최적의 도구를 조합해 사용하는 **Headless Architecture(마이크로 프론트엔드/백엔드 분리)** 전략을 채택/의결합니다.
- **프론트엔드 분할 정책:** 
  - 검색 최적화(SEO)와 빠른 초기 렌더링이 필수인 '웹(대문 베이스캠프)'은 생태계의 표준인 **Next.js (React 기반)**를 사용합니다.
  - 디바이스 하드웨어에 접근하거나 유려한 네이티브 앱 경험이 중요한 모바일 중심 모듈(파이낸스, 라이프 로그)은 기존 강점을 살려 **Flutter**를 사용합니다.
- **백엔드 코어 (API/DB):** 
  - 프론트엔드의 트렌드가 변하더라도 영원히 생존할 수 있는 통합 **RESTful API** 서버 코어를 **Java(Spring Boot 생태계)** 기반으로 구축합니다. 이는 "시간이 다소 걸리더라도 진정한 엔터프라이즈급 백엔드 아키텍처를 설계하여 강력한 코어로 만든다"는 설계자의 의지를 반영합니다.

## Consequences
- **Positive:** 
  - 각 플랫폼(Web/App)이 최상의 퍼포먼스를 냅니다.
  - 프론트엔드가 교체되더라도 핵심 백엔드 자산(`Legacy_Core`)과 강력한 비즈니스 로직은 영구 보존됩니다.
  - 글로벌 메이저 IT 기업에서 선호하는 Java 기반 묵직한 백엔드 아키텍처 구성 이력이 포트폴리오로 증명됩니다.
- **Negative:** 
  - 여러 언어 프레임워크(Dart/TS/Java) 환경을 1인이 모두 관리해야 하므로 Context Switching 피로도가 높고 초기 세팅 비용이 증가합니다. (단, 이는 극복의지 및 SRE 에이전트의 보조로 상쇄합니다.)
