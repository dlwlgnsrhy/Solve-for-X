# 2. 로컬 LLM 모델 선정 (Qwen3-Coder 30B)

Date: 2026-03-28

## Status
Accepted

## Context
- 외부 클라우드 의존성 없이 프라이버시를 보호하면서 '레거시 설계자'의 코딩 능력을 극대화할 수 있는 로컬 엔진이 필요했습니다. (LM Studio 활용)
- 초기 다운로드 용량(8B 등 가벼운 모델)과 실제 작업 시 추론 성능(30B 이상 무거운 모델) 사이의 선택이 필요했습니다.
- 프로젝트의 핵심 가치는 '기술적 완성도'와 '이지훈의 정체성 구현'에 있으므로, 단순한 속도보다는 정교한 결과물이 우선시되었습니다.

## Decision
- 용량과 초기 로딩 시간의 손해를 감수하더라도, 작업 품질이 프로젝트 성공의 핵심이므로 **Qwen3-Coder 30B**를 기본 모델로 채택하였습니다.
- 성능이 용량보다 중요하다는 설계자님의 판단을 최우선으로 반영하였습니다.

## Consequences
- **Positive:** 높은 수준의 코드 이해 및 생성 능력 확보로 SRE 에이전트 작업 효율성 증대.
- **Negative:** 고사양의 하드웨어 리소스(VRAM 등) 요구, 최초 구동 시 모델 로딩 시간 증대.
