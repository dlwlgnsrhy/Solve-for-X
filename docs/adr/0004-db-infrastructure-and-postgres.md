# ADR 0004: DB 인프라 설계 및 PostgreSQL 채택 전략 (비용 최적화 관점)

**상태 (Status):** 논의 중 (In Discussion)  
**날짜 (Date):** 2026-04-02  
**관련 로드맵:** Phase 2 (Legacy_Core Engine)

---

## 1. 배경 및 목적 (Context)
SFX(Solve-for-X) 생태계의 Phase 2에 진입함에 따라, 데이터를 영구적으로 저장할 메인 데이터베이스(DB) 인프라 구축이 필요해졌습니다. 이 DB는 추후 개인 자산 트래킹 데이터 및 생애 서사(Life-Log) 등 다목적 레코드를 감당해야 하므로 안정성과 유연성이 필수적입니다. 본 문서는 **왜 PostgreSQL을 사용해야 하는지**와 **서버를 어디에(로컬 vs 클라우드) 띄울 것인지**에 대한 비용 최적화 관점의 논의를 기록합니다.

---

## 2. 의사결정 1: 왜 PostgreSQL인가? (확장성 측면)

다양한 RDBMS(MySQL, MariaDB, Oracle)와 NoSQL(MongoDB) 중 PostgreSQL을 기반으로 설계하는 이유입니다.

### 👍 장점 및 확장성 시나리오
1. **정형 데이터와 비정형 데이터의 하이브리드 처리 (JSONB)**
   - 자산/금융 데이터(Finance)는 강력한 트랜잭션과 무결성이 필요한 전형적인 '정형 데이터'입니다. 
   - 반면 Life-Log(Memory)는 메타데이터가 텍스트, 이미지 태그, 음성 길이 등 아주 다양합니다. PostgreSQL의 `JSONB` 컬럼을 사용하면 MySQL이나 MongoDB를 별도로 쓰지 않고도 NoSQL의 유연함을 그대로 흡수할 수 있습니다.
2. **AI 에이전트 확장성 (pgvector)**
   - 향후 로컬 LLM이 일기나 생애 서사를 '의미 기반 검색(Semantic Search)' 하려면 벡터 DB가 필요합니다. PostgreSQL은 `pgvector` 확장을 통해 단일 DB 내에서 완벽한 AI 벡터 검색을 지원합니다.
3. **가장 완벽한 오픈소스 생태계**
   - 글로벌 SRE/SWE 기준, 현재 가장 트렌디하고 강력한 커뮤니티 지원을 받는 관계형 DB이며 구글, AWS 등 모든 클라우드 호환성이 압도적으로 높습니다.

---

## 3. 의사결정 2: Cloud vs Local 인프라 비용/SRE 최적화 비교

초기 1인 개발/운영 체제에서 엔터프라이즈급 인프라를 처음부터 도입하면 클라우드 비용(Monthly Cost)이 수익을 갉아먹을 위험이 있습니다.

### 옵션 A. 100% Cloud (AWS RDS / Supabase / Neon)
- **작동 방식:** 클라우드 서비스에 DB와 Spring Boot 서버를 띄워 24시간 가동.
- **장점:** 
  - SRE 관점에서 99.9% Uptime이 보장되며 무중단 모니터링이 가능.
  - 노트북 전원이 꺼져도 외부(Flutter 모바일 앱 등)에서 언제든 접속 가능.
- **단점 (비용):** 
  - AWS EC2(또는 ECS) + RDS 사용 시, 가장 싼 것만 써도 **월 $30~$50 (약 4~7만 원)** 이상의 고정비용 발생.
  - 트래픽이 없는 개발 초기 단계에선 100% 매몰 비용(Sunk Cost)이 됨.
- **대안 (Serverless Free-Tier):** Supabase나 Neon 등 Serverless Postgres 무료 티어를 쓰면 DB 비용은 0원이지만, 용량(500MB 등)이나 커넥션 제한이 매우 빡빡함.

### 옵션 B. 100% Local + 터널링 (MacBook / 홈 서버 + Docker)
- **작동 방식:** 현재 코딩하고 계신 설계자님의 Mac(혹은 남는 노트북)에 Docker로 DB와 서버를 띄우고, `Cloudflare Tunnels`나 `Ngrok`을 통해 외부망과 연결.
- **장점:** 
  - **비용 완벽한 $0.** AWS 트래픽 요금이나 DB 유지비가 전혀 없음.
  - 내 맥북의 SSD(수백 GB)와 RAM을 마음껏 끌어다 쓸 수 있어 초기 성능이 압도적임.
- **단점 (운영 리스크):** 
  - 노트북 뚜껑이 닫혀 슬립 모드에 빠지거나 와이파이가 끊기면 **서버가 다운(SRE 레드 얼럿)** 됨.
  - 외부에서 Flutter 모바일 앱으로 다이어리를 쓰려 할 때, 맥북이 집에 켜져 있어야만 작동함.

### 옵션 C. 하이브리드 아키텍처 (점진적 마이그레이션) ★강력 추천
1. **[초기 개발 단계]: 100% Local Docker (비용 $0)**
   - 현재는 유저가 설계자님 1명뿐이며 개발 & 디버깅이 주 목적입니다. 로컬 맥북에 Docker Compose로 PostgreSQL을 띄워 무료로 빠르게 개발합니다.
2. **[모바일 앱 배포 단계]: Cloud Free-Tier 결합 (비용 $0)**
   - Spring Boot 서버는 클라우드 무료 서버(AWS EC2 Micro, Oracle Cloud 무료 티어, Koyeb 등)에 띄워 24시간 유지시킵니다.
   - DB는 개발 생산성이 높고 무료인 Supabase(혹은 Neon Database)를 연결해 외부 접속을 뚫습니다.
3. **[스케일업 단계]: AWS 전환 (비용 발생 시작)**
   - 데이터가 500MB를 넘거나, 트래픽이 유의미해지면 그때 가서 DB를 AWS RDS 등 본격 유료 인프라로 이전합니다.
