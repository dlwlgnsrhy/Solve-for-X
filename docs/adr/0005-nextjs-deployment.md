# ADR-0005: Next.js 배포 아키텍처 결정 (비용 최적화 및 학습 극대화)

**작성일**: 2026-04-03
**안건**: 비용(Cost)을 제로(0)에 수렴시키고, SRE/엔지니어링 학습(Learning) 성장은 극대화하는 `Legacy_Core` 베이스캠프 배포 전략 확립

## 1. 배경 (Context)
초기 제안된 아키텍처 중, 설계자님의 핵심 강점인 '학습에 대한 열망'을 자극하면서도 단점인 '금전적 비용'을 완벽하게 방어할 수 있는 최저 비용 옵션에 대한 딥다이브(Deep-Dive)가 요구되었습니다. 

## 2. 최저 비용(Zero-Cost) 대안 상세 분석

### 대안 A: Oracle Cloud Free Tier + 자가 호스팅 (학습치 MAX, 비용 $0)
오라클 클라우드는 평생 무료(Always Free)로 **ARM 기반 4코어 OCPU, 24GB RAM**의 엄청난 서버를 제공합니다. AWS EC2 프리티어(1GB RAM)와 비교조차 안 될 정도로 무인 공장 용도로는 최고의 하드웨어입니다.
- **아키텍처**: 무료 우분투(Ubuntu) 서버를 할당받아, Docker를 설치하고, Next.js 이미지를 빌드하여 띄운 뒤, Nginx로 리버스 프록시와 SSL 암호화를 씌웁니다.
- **학습 곡선**: 리눅스 커널, 네트워크 포트 개방, 리버스 프록시, Docker Compose, 수동 CI/CD 등 모든 SRE 기본기를 맨바닥에서 구를 수 있습니다. (진짜 '레거시 설계자'의 길)
- **비용**: 100% 무료 (트래픽이나 서버비 없음)
- **단점**: 오라클 프리티어 계정 가입이 다소 깐깐함(카드 튕김 현상 잦음). 초기 세팅 시 며칠 밤을 새울 확률이 높음.

### 대안 B: Cloudflare Pages (네트워크 지식 습득, 비용 $0)
Vercel을 대체할 가장 이상적인 글로벌 CDN 인프라입니다. 과도한 트래픽이 몰려도 비용 폭탄(DDOS 과금)을 막아내는 세계 1위 CDN 기업의 서비스입니다.
- **아키텍처**: 깃허브 저장소와 연동 시, Cloudflare가 코드를 가져가서 Edge(전 세계 캐시 서버) 단위로 쪼개어 배포합니다.
- **학습 곡선**: Cloudflare Workers 인프라, `edge` 런타임 제약사항, 전 지구적 CDN 캐싱 메커니즘을 경험할 수 있습니다.
- **비용**: 완전 무료 (대역폭 요금 무제한 0원)
- **단점**: Next.js의 기존 순수 API Route(Node.js 환경)와 100% 호환되지 않아, 코드를 `@cloudflare/next-on-pages`용으로 변환하거나 구조를 조금 고쳐야 하는 런타임 제약이 있습니다.

### 대안 C: AWS EC2 t2.micro Free Tier (초보적 클라우드 SRE, 1년간 $0)
오라클 클라우드 가입 실패 시의 가장 무난한 대안입니다.
- **아키텍처**: AWS 계정 생성 직후 1년 동안 무료로 제공되는 1GB RAM 우분투 서버.
- **학습 곡선**: 대안 A와 동일. 하지만 인스턴스 성능이 아주 낮아 Swap 메모리 설정 등 "가난한 서버 살려내기" 기법 연구가 병행됨.
- **비용**: 1년간 무료 (단, 이후 과금 전환). 대역폭(Egress)이 월 100GB 초과 시 과금 발생 주의.
