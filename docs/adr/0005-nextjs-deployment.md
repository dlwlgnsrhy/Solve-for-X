# ADR-0005: 요금 폭탄 원천 차단 및 무영구(Zero-Cost) 인프라 확정

**작성일**: 2026-04-03
**상태**: `결정 완료 (Approved)`
**안건**: 구글 클라우드(GCP) 과금 사태의 반면교사를 토대로, 다중 앱 Lean 테스트를 위한 무과금 배포망(인프라) 확정.

## 1. 근본 원인 분석: GCP의 1GB 아웃바운드 함정
과거 10만 원 과금이 발생했던 원인은 GCP 프리티어의 **네트워크 아웃바운드 1GB 제한** 조항 때문입니다. 
GCP는 악명 높게도 한도를 초과하면 서비스 차단이 아닌 **'즉시 신용카드 청구'**를 진행합니다. 이는 사용자 없는 소규모 앱 여러 개를 띄우고 테스트하는 설계자님의 스타트업 방식에 치명적인 독약입니다.

## 2. 최종 오라클 클라우드 프리티어 선정의 당위성
NotebookLM AI 교차 검증 및 전체 아키텍처 토론 결과, 오라클 클라우드(Oracle Cloud)의 상시 무료(Always Free) 티어가 모든 제약 조건을 무효화하는 유일한 해답으로 채택되었습니다.

1. **GCP의 1만 배 트래픽**: 10,000GB(10TB)의 아웃바운드 무료 한도.
2. **비용 하드 컷(Hard-Cut)**: 오라클은 사용자가 스스로 "종량제 업그레이드" 버튼을 누르지 않는 이상, 과금 트리거 자체를 발생시키지 않고 서버를 중지시킴.
3. **가공할 하드웨어**: 무려 **24GB RAM / ARM 4코어**라는 말도 안 되는 스펙. 이정도면 깡통 서버 1대에 10개의 브랜드 맵과 SRE 파이썬 봇, Spring Boot 코어를 모두 Docker로 욱여넣을 수 있습니다.

## 3. 유휴 인스턴스 회수(Reclaim) 정책 방어책
오라클 클라우드의 유일한 치명적 단점인 "사용량이 없으면 서버 삭제" 정책을 방어하기 위해 다음 조치를 취하기로 문서화합니다.

*   서버에 `NeverIdle` 같은 부하 발생 봇을 돌리거나, 아니면 과감히 **종량제 업그레이드 활성화** 후 오직 **"항상 무료(Always Free)" 리소스 1개만 사용**하여 회수 규칙을 회피합니다. (이때 요금은 0원 유지됨)
*   GitHub 및 S3 정기 백업 파이프라인 의무화.

## 4. 최종 아키텍처 결론
> **Oracle Cloud VPS (ARM 24GB) + Coolify (자가 호스팅 PaaS) + Github Actions**
- 향후 모든 프로젝트(프론트/백엔드/DB)는 이 무인 공장의 컨테이너 레이어 속으로 통합되어 배포됩니다.
