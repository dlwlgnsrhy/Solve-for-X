# 📋 노션 일간 계획 템플릿

> 매일 아침 이 형식으로 노션에 작성 → 텔레그램에 `/start` 전송
> 에이전트가 이 페이지를 읽고 실행 브리핑 + 작업 시작

---

## ✍️ 작성 방법

노션에서 새 페이지를 만들고 아래 형식을 그대로 복붙해서 채우면 됩니다.
**핵심: 간결하게, 빠르게. 5분 이상 쓰지 않는다.**

---

## 📄 템플릿 (노션에 복붙)

```
📅 YYYY-MM-DD 오늘의 계획

오늘의 1가지: [딱 하나, 오늘 반드시 끝낼 것]
컨디션: [1-10]
포커스: [코드 / 문서 / 전략 / 인프라 / 혼합]

━━━━━━━━━━━━━━━━━━━━

- [ ] [Top 1 — 가장 중요한 것]
- [ ] [Top 2]
- [ ] [Top 3]

에이전트에게 위임:
- [에이전트가 자율 실행해도 되는 것]
- [에이전트가 자율 실행해도 되는 것]

오늘 하지 않을 것:
- [의도적으로 미루는 것]
```

---

## ✅ 작성 예시

```
📅 2026-04-29 오늘의 계획

오늘의 1가지: Imjong Care 인증 화면 완성
컨디션: 8
포커스: 코드

━━━━━━━━━━━━━━━━━━━━

- [ ] 카카오 로그인 SDK 연동
- [ ] 로그인 화면 UI 구현
- [ ] 단위 테스트 작성

에이전트에게 위임:
- flutter analyze 실행 후 오류 자동 수정
- README.md 인증 섹션 업데이트

오늘 하지 않을 것:
- 배포 파이프라인 설정 (다음 주)
- 새 기능 추가 (인증 먼저)
```

---

## 🔑 핵심 규칙

| 규칙 | 이유 |
|------|------|
| **5분 안에 작성** | 계획에 에너지를 쓰지 않고 실행에 쓴다 |
| **Top 3 이하** | 3개 이상은 현실적으로 불가능 |
| **"에이전트 위임" 명시** | 에이전트가 뭘 자율 실행해도 되는지 명확히 |
| **"하지 않을 것" 명시** | 집중력 보호, 에이전트도 해당 작업 건드리지 않음 |
| **작성 완료 → `/start` 전송** | 트리거가 명확해야 에이전트가 실행 |

---

## 📱 저녁 루틴 (작성 불필요)

저녁에는 별도 기록이 필요 없습니다.
에이전트가 자동으로:
1. 오늘 Git 커밋 + 오늘 페이지 분석
2. 전문가 피드백 + 내일 제안 생성
3. 노션 페이지에 리포트 추가
4. 텔레그램으로 발송

당신은 받은 리포트 읽고 → `/feedback [지시]` 또는 `/done` 만 입력하면 됩니다.
