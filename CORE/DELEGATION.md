# 🧭 에이전트 위임 정책

> **핵심 원칙**: 나(지훈)는 방향을 결정하고 에이전트는 실행한다.
> 에이전트는 나의 계획을 읽고 일을 하며, 결과를 나에게 보고한다.

---

## 🔁 하루 루틴 구조

### 🌅 아침 (07:00~08:00) — 방향 선언

| 주체 | 행동 |
|------|------|
| **나** | 노션 "오늘의 계획" 페이지 작성 (목표, Top 3, 컨디션) |
| **나** | 텔레그램에 `/start` 전송 → 에이전트에게 시작 신호 |
| **에이전트** | 노션 오늘 페이지 읽기 → 태스크 파싱 → 실행 시작 |
| **에이전트** | "오늘 계획 확인. 작업 시작합니다 ✅" 응답 |

### ☀️ 낮 (에이전트 단독 실행)

| 주체 | 행동 |
|------|------|
| **나** | 핵심 업무에만 집중 (에이전트에게 맡긴 것은 신경 끄기) |
| **에이전트** | 계획 기반 자율 실행 (코드, 문서, CI, 테스트 등) |
| **에이전트** | 완료/이슈 발생 시 텔레그램 중간 보고 |
| **에이전트** | 18:00에 일일 리포트 자동 생성 및 전송 |

### 🌆 저녁 (18:00~19:00) — 확인 & 피드백

| 주체 | 행동 |
|------|------|
| **에이전트** | 오늘 완료된 작업 전체 정리 → 노션 + 텔레그램 리포트 |
| **나** | 리포트 확인 후 텔레그램으로 짧게 피드백/지시 |
| **에이전트** | 피드백 수신 → 개선 작업 즉시 실행 → 완료 보고 |
| **나** | 중요 결정만 CORE 문서 업데이트 (5분) |

---

## 💬 텔레그램 커맨드 정의

| 커맨드 | 설명 | 에이전트 반응 |
|--------|------|--------------|
| `/start` | 아침 루틴 시작 | 노션 오늘 계획 읽기 → 실행 시작 |
| `/status` | 현재 진행 상황 확인 | 완료/진행중/대기 태스크 목록 출력 |
| `/report` | 즉시 리포트 요청 | 지금까지 완료된 작업 즉시 정리 |
| `/done` | 오늘 마무리 | 최종 리포트 생성 + 내일 준비 항목 제안 |
| `/feedback [내용]` | 피드백 전달 | 피드백 반영 후 작업 조정 및 실행 |

## 🤖 에이전트 자율 실행 범위

### ✅ 자율 실행 가능 (승인 불필요)
- 코드 린트, 포맷, 정적 분석
- 단위 테스트 실행 및 커버리지 확인
- 문서 초안 자동 업데이트
- Git 커밋 내역 분석 및 요약
- 의존성 버전 확인 및 업데이트 제안
- 노션 일간 리포트 작성

### ⚠️ 제안 후 내 승인 필요
- PR 생성 및 머지
- 배포 파이프라인 트리거
- 외부 API 연동 변경
- 비용 발생 작업

### 🧠 나만 결정
- 제품 방향성 및 기능 우선순위
- 아키텍처 전환 결정
- 협업·파트너십
- CORE 문서 내용 변경

## 📅 위임 신뢰 확장 단계

| Phase | 기간 | 에이전트 자율 범위 |
|-------|------|-----------------|
| **Phase 1** | 지금 | 제안만, 실행은 내 승인 |
| **Phase 2** | 1개월 후 | 낮은 위험 작업 자율 실행 (린트, 문서, 테스트) |
| **Phase 3** | 3개월 후 | 중간 위험 작업 자율 실행 (PR 생성, 이슈 등록) |
| **Phase 4** | 이후 | 전체 파이프라인 자율 운영, 나는 방향만 결정 |
